to access the logs of the docker images:
docker compose logs cleartax_nodejs

this command - while you are inside the server folder creates a 1000 users
docker exec -it cleartax_nodejs node --experimental-strip-types src/seedUsersRegular.ts

this command - while you are inside the server folder create 1000 transactions
docker exec -it cleartax_nodejs node --experimental-strip-types src/seedTransactions.ts

this command - while you are inside the server folder - it creates a dozen of company users
docker exec -it cleartax_nodejs node --experimental-strip-types src/seedUsersCompany.ts

this command - while you are inside the server folder create a dozen fake data points for the table
docker exec -it cleartax_nodejs node --experimental-strip-types src/seedProblems.ts

####################### postgres
1. Open the Registration Dialog
Log in to pgAdmin at http://localhost:5050.

Right-click on Servers in the left-hand browser menu.

Select Register > Server....

2. General Tab
Name: Give it any name you like (e.g., ClearTax_Dev).

3. Connection Tab (Crucial Step)
This is where you link the two containers. Do not use localhost here; use the Docker service name.

Host name/address: postgres (This matches the service name in your YAML file).

Port: 5432

Maintenance database: Use the value of your ${DB_NAME} (usually the name of your app database).

Username: Use the value of your ${DB_USER}.

Password: Use the value of your ${DB_PASSWORD}.

Save Password: Check this so you don't have to re-enter it every time.

Click Save.
